位运算

  • 正数直接进行位操作
  • 负数需先转成补码，然后进行位操作

1. &操作（与）

  • 运算规则：全1则1，否则为0

  • 例子：1&1 = 1，1&0 = 0， 0&0 = 0

  • 作用：

    1. 用来将数归零，通过将该数与0进行&操作
    2. 确保某个数的任意位是0，通过将该数与该位为0的数进行&操作

2. | 操作（或）

  • 运算规则：全0则0，否则为1
  • 例子：1|1 = 1， 1|0 = 1， 0|0 = 0
  • 作用：
    确保某个数的任意位是1，通过将该数与该位为1的数进行|操作

3. ^操作（异或）

  • 运算规则：相同则0，不同则1
  • 例子：0^0 = 0, 1^1 = 0, 0^1 = 1
  • 作用：
    1. 与0进行异或操作，得到的结果是本身
    2. 确保某个数的任意位是反转的，通过将该数与该位为1的数进行^操作

4. <<操作（有符号左移）

  • 运算规则：全部向左移一位，高位丢弃，低位补0
  • 例子： 2<<1 = 4， -2<<1 = -4
  • 作用：若高位的1没有被丢弃，则相当于与2^n相乘

5. >>操作（有符号右移）

  • 运算规则：全部向右移一位，正数高位补0，负数高位补1
  • 例子： 4>>1 = 2, -4>>1 = -2
  • 作用：若低位的1没有被丢弃，则相当于与2^n相除

6. 不同长度的数进行位运算

  • 运算规则：按右边对齐，长度小的数进行高位补充
  • 若是正数，则高位补0；若是负数，则高位补1

一、redis的简介

$\qquad$ redis是一个键值型的数据库，key一般是个string，而value则有很多种类型。

1. redis的key结构

因为key是不可重复的，但redis又没有table这种结构，那么当多个数据对应的id相同时，该怎么处理？

因此，redis允许key有多个单词形成层级结构，多个单词之间通过‘:’的形式隔开，即

$$项目名：业务名：类型：id$$

2.redis的value结构

其中较为常用的为：

  • string：字符串类型 -> “hallo world”
  • hash：键值对 -> {name: “LiCan”, password : 123456}
  • List：列表 -> {A -> B -> C}
  • Set：集合 -> {A, B, C}
  • SortedSet：有序集合 -> {A: 1, B: 2, C: 3}

二、redis的基本使用

1.redis的登录

redis-cli -h 127.0.0.1 -p 6379 -a password

  • -h：要登录的redis所在的IP，默认为本地
  • -p：redis对外的端口，默认为6379
  • -a：redis的密码，没有就默认不填

2.redis的通用命令

  1. 添加一条新数据： set key value
    • key为键，如果没有会报错
    • value为实际值，如果没有会报错
    • 例：set name “LiCan”
  2. 获取一条已存在的数据： get key
    • key为已存在的值。如果是不存在的值，则返回结果为空
    • 例：get name -> 结果为 “LiCan”
  3. 根据正则表达式获取key： keys pattern
    • pattern为正则表达式
    • 例：keys * -> 代表查看所有已存在的key
  4. 删除已存在的数据： del key [key ···]
    • key为键，此处可以有多个key
    • 例：del name -> 结果返回：(integer) 1 ，代表已删除一条数据
  5. 批量新增数据： mset key value key value ···
    • key为键，如果没有会报错
    • value为实际值，如果没有会报错
    • 例：mset name “LiCan” password 123456
  6. 为数据设置过期时间： expire key time
    • key为键，如果没有会报错
    • time为时间，单位为秒
    • 例：expire name “LiCan” 10 -> 表示为这条以name为key的数据设置10秒的存在期，到期直接删除
  7. 查看数据的到期时间： ttl key
    • key为键，如果没有会报错
    • 如果查看的key是永久的，则结果是 -1
    • 如果查看的key被设置了过期时间，则结果有两种。一种是还没到期，则是剩余时间；另一种是已经到期，则是 -2
    • 查看不存在的key，则结果是 -2

3. String类型

1. 存储类型分类

该类型的value是字符串，但根据字符串的格式不同，又分为3类：

  • 普通字符串，例：abcd
  • 整型，例：123
  • 浮点类型，例：24.6

2. 常用命令

  1. 添加一条新数据： set key value，如果key已存在，则会覆盖之前的内容
  2. 获取一条数据： get key
  3. 添加多条新数据： mset key value [··· key value]
  4. 获取多条数据： mget key [··· key]
  5. 自增操作： incr key，默认自增1，前提是整型或浮点型
  6. 自减操作： decr key，默认自减1，前提是整型或浮点型
  7. 自增指定步长： incrby key increment，increment为自增步长
  8. 浮点自增操作： incrbyfloat key increment，此处的increment可以是小数
  9. 真正的新增操作： setnx key value，如果key已存在，则不会新增
  10. 在新增数据时添加过期时间： setex key time value，time的单位为秒

4.Hash类型

1. 简介

该类型的value类似于Java中的HashMap，是个键值对。常用于存储对象以及修改对象属性值。

2. 常用命令

  1. 添加一条新数据： hset key field value，如果key或field已存在，则会覆盖之前的内容
  2. 获取一条数据： hget key
  3. 向同一个key中添加多条新数据： hmset key field value [··· field value]
  4. 向同一个key中获取多条数据： hmget key field [··· field]
  5. 自增指定步长： hincrby key field increment，increment为自增步长
  6. 浮点自增操作： hincrbyfloat key field increment，此处的increment可以是小数
  7. 真正的新增操作： hsetnx key field value，如果field已存在，则不会新增
  8. 获取一个key中所有的field： hkeys key
  9. 获取一个key中所有的value： hvals key
  10. 获取一个key中所有的field和value： hgetall key，结果是奇数项是field，偶数项是value

5.List类型

1.简介

该类型的value类似于Java中的linkList，是个双向链表。因此，插入和删除速度快，查询效率一般。且可用来构成队列和栈

2. 常用命令

  1. 添加一条新数据： lpush(左插入) / rpush(右插入) key element [··· element]
  2. 获取一条数据： lpop(左取出) / rpop(右取出) key
  3. 返回范围内的所有元素： Lrange key begin end，begin是开始下标，end是结束下标，两边都是闭空间。
  4. 获取一条数据，若没有则阻塞： brpop / blpop key timeout，timeout是阻塞最大时间。

6.Set类型

1.简介

该类型的value类似于Java中的hashset，可以看作一个hashmap，但value皆为null。因此，其具备：

  • 无序
  • 查询速度快
  • 不可重复

2. 常用命令

  1. 新增一条新数据： sadd key member [··· member]
  2. 获取所有数据： smembers key
  3. 移除指定数据： srem key member [··· member]
  4. 判断某元素是否存在： sismember key member
  5. 返回当前集合中元素的总数： scard key
  6. 求key1与key2的交集： sinter key1 key2
  7. 求key1与key2的差集： sdiff key1 key2，即存在于key1集合中，但不存在于key2集合中的元素
  8. 求key1与key2的并集： sunion key1 key2

7.sortedSet

1.简介

该类型的value仍是hashSet，但多了一个score，用来排序。即底层存储仍是无序的，但返回结果前会根据score进行排序

2. 常用命令

  1. 新增数据： zadd key score member [··· score member]
  2. 移除指定数据： zrem key member [··· score member]
  3. 获取集合中某元素的score值： zscore key member
  4. 获取集合中某元素的排名： zrank key member，默认是从小到大排；若相反，则在z的后面加上rev
  5. 获取集合中元素的总数： zcard key
  6. 获取指定范围内的元素总数： zcount key begin end，默认是从小到大排；若相反，则在z的后面加上rev
  7. 获取指定score值范围内的元素： zrangebyscore key beginScore endScore
  8. 获取指定范围内元素： zrange key begin end，默认是从小到大排；若相反，则在z的后面加上rev
  9. 自增集合中某元素的score： zincrby key increment member
  10. 求key1与key2的交集： zinter key1 key2
  11. 求key1与key2的差集： zdiff key1 key2，即存在于key1集合中，但不存在于key2集合中的元素
  12. 求key1与key2的并集： zunion key1 key2

三、redis的使用场景

  1. 计数器： 可以对String进行自增自减运算，很适合存储频繁读写的计数量
  2. 缓存： 将数据放到内存中，设置内存的最大使用量以及淘汰策略来保证缓存的命中率
  3. 会话缓存： 将用户与应用服务器的会话信息（如：token）存储在内存中，形成有时效的登录令牌
  4. 消息队列（发布/订阅功能）： List类型是个双向链表，可以通过lpush和rpop来写入和读取消息。也可以配合Kafka、RabbitMq等消息中间件实现完整的功能。

四、过期键的删除策略

Redis有三种不同的删除策略：立即删除，惰性删除，定时删除

  • 立即删除。在设置键的过期时间时，创建一个回调事件，当过期时间达到时，由时间处理器自动执行键的删除操作。
  • 惰性删除。键过期了就过期了，不管。当每次按key取值时，先检查此key是否已经过期，如果过期，则删除，并返回nil，如果没过期，就返回键值。
  • 定时删除。每隔一段时间，对expires字典进行检查，删除里面的过期键。

以上三种中，第二种是被动删除，其他是主动删除，第一种的实时性最高。下面对三种策略进行具体分析：

  • 立即删除
    立即删除能够保证内存的利用率最高。但对CPU是最不友好的，因为删除操作会占用CPU时间，如果刚好碰上了CPU很忙的时候，
    比如正在做交集或排序等计算的时候，就会给CPU造成额外的压力。
  • 惰性删除
    当键值过期后，不会立即被删除。直到按key取值时，才会被检查到过期，然后被删除。所以惰性删除对内存的占用最高。
  • 定时删除
    从上面的分析来看，立即删除会短时间内占用大量CPU，而惰性删除则会在一段时间内占用大量内存。因此定时删除是个折中的办法。
    定时删除是：每隔一段时间进行一次删除，一方面减少了删除的频率，从而减少对CPU的占用。另一方面减少了因惰性删除带来的内存浪费。

ps：redis使用的策略是：惰性删除加上定时删除，两者配合使用。

1.事务简介

1.ACID特性

  • 原子性：事务要么全做，要么全不做
  • 一致性：事务必须是从一个一致性状态变到另一个一致性的状态
  • 隔离性：事务与事务之间相互独立，互不干扰。
    隔离级别：
    1.读未提交：可以读取别人未提交的数据
    2.读已提交：只能读取到别人已经提交的数据。可以防止脏读
    3.可重复读：采取你第一次读取到的数据，此后，其他人不准修改。可以防止不可重复读
    4.可串行化：你在进行增删改查期间，其他人只能等着。可以防止幻读
  • 持久性：一旦事务提交，它对数据库的修改是永久的。

2.并发事务的问题

  • 脏读：你读取到的数据是错误的。你第一次读数据后，别人进行了修改(但未提交)，然后你用这错误的数据进行了操作。
  • 不可重复读：你读取到的数据，前后不一致。但读取到的数据是正确的，只是别人进行了修改。
  • 幻读：你读取到的数据量不一致。但读取到的数据量是正确的，只是别人进行了新增和删除。

2.单体事务用法

$\qquad$在SpringBoot中使用单体事务需要用到@Transactional注解，依赖包为

org.springframework.transaction.annotation

__注意__：声明该注解的方法应放在service层，因为该方法需要成为动态代理对象，而controller层不是组件(Component)

4.分布式事务协议2PC

目的：为解决多个服务之间保持事务的一致性。
$\qquad$对象分为两个部分，一个是协调者，一个是参与者(可以有多个)。参与者可以看作是微服务，也可以看作数据库

1.两个阶段：

  1. 预处理阶段prepare：
    1. 协调者向所有的参与者发送事务请求，然后等待各个参与者的响应
    2. 参与者执行事务操作(但不提交)，并将undo和redo信息记录在事务日志中
    3. 如果参与者执行事务成功并写入undo和redo信息，则向协调者返回YES，否则返回NO。当然，也可能宕机，什么都不返回
  2. 提交阶段commit：
    1. 如果协调者收到所有参与者的YES信号，那么向所有参与者发送提交请求，参与者进行事务提交,然后释放资源。
    2. 参与者向协调者发送Ack反馈响应。接收到所有的Ack相应后，完成一次事务提交。
    3. 如果没有收到全部的YES信号，那么协调者会进入回退流程，对事务进行undo操作。
    4. 参与者向协调者发送Ack反馈响应。接收到所有的Ack相应后，完成一次事务回退。

2.2PC的问题

  • 同步阻塞。其他参与者需要等待未完成响应的参与者，而无法进行其他操作。若参与者与协调者之间网络异常，导致参与者收不到协调者的信息，那么会导致参与者一直阻塞下去。
  • 协调者的地位太过重要，一旦协调者宕机，那么参与者会一直阻塞。
  • 因为网络原因或协调者宕机，可能让部分参与者没有收到请求而一直阻塞，但其他参与者却执行了操作，这样数据就不一致了。补救措施就是向程序员发送不确定警告。

5.AT模式（Auto Transaction）

$\qquad$AT模式是一种无侵入的分布式事务解决方案。阿里seata框架，实现了该模式。
在AT模式下，用户只需关心自己的“业务SQL”，用户的“业务SQL”作为一阶段，Seata框架自动生成事务的二阶段提交和回滚操作。

  1. 一阶段：Seata会拦截“业务SQL”，首先解析SQL语义，找到“业务SQL”要更新的数据。在数据被更新前，将其保存为“before image”，然后执行“业务SQL”。在业务数据更新后，再将其保存为“after image”，最后在提交前请求行级锁，若成功则提交，若失败，则回滚。

  1. 二阶段：
  • 如果全部执行完成，那么就直接将“before image”和“after image”删除，然后解除行级锁。一次事务结束。
  • 如果没有全部执行，那么进行回滚。
    1. 先进行校验脏写。对比“数据库当前业务数据”和“after image”，如果一致则还原数据，否则转人工处理。
    2. 借由“before image”生成逆向SQL，进行数据还原。
    3. 删除中间数据，解除行级锁。

6.TCC模式（Try Confirm Cancel）

$\qquad$TCC模式是一种侵入性较强的分布式事务解决方案。事务的执行、提交和回滚内容全由程序员自己实现。

  1. 一阶段：事务发起者执行Try方式。事务请求来临后，执行中间临时数据的修改。然后向中间协调者发送成功响应。
    例：当下单请求来临后，我可以把订单服务的status从“未支付”改为“支付中”。
  2. 二阶段：
    1. 事务发起者成功执行一阶段。然后执行Confirm方式，执行真正的数据的修改。
      例：将订单服务的status从“支付中”改为“已支付”。
    2. 事务发起者执行失败（Try和Confirm任何一方失败）。执行Cancel方式，回滚操作。
      例：将订单服务的status从“已支付”改为“未支付”。

Lamda表达式

$\qquad$Lamda表达式是一种函数式编程方式，主要是用来简化函数式接口的匿名内部类的写法。

1.使用条件

  • 用于new匿名内部类
  • 该匿名内部类实现的接口是函数式接口，即只有一个抽象方法需要实现

2.基本写法

$\qquad$去掉对象和方法名，然后在形参和方法体之间加上->

举个例子：
原来的Java写法为

int[] users = {2,1,3}
array.sort(users, new Comparator(
    @override
    public int compare(int o1, int o2) {
        return o1 - o2;
    }
));

Lamda表达式的写法为：

int[] users = {2,1,3}
array.sort(users, (int o1, int o2)->{
        return o1 - o2;
    });

3.可推导，可省略

  1. 参数类型可以省略（根据形参的位置可以得知参数的类型）
  2. 如果参数只有一个，()也可以省略（如果有多个参数却省略的话，就不知道前面的几个参数是输入的实参，还是Lamda表达式中的形参了）
  3. 如果函数体只有一行，return、{}和;可以一起省略（注意：要么一起省略，要么都不省略）

举例：
将上述Lamda表达式继续简化为

int[] users = {2,1,3}
array.sort(users, (o1, o2)->
        return o1 - o2;
    );

序言

记录自己学习Markdown标记语言的学习内容。

一、标题

以’#’开头表示标题。一个#对应HTML中的<p1>标签，以此类推。

例子：

输入 # test后的结果显示为

输入 ## test后的结果显示为

二、段落

$\qquad$简单写一行就是一个段落。
换行方法：

  • 在要换行的位置空两个空格
  • 敲两次回车

三、字体样式

  • 粗体：使用**或__
  • 斜体：使用*或_
  • 联合使用：使用**_或__*

四、图片

通过使用![图片描述](图片地址（相对地址）)来显示图片。

五、列表

  • 以数字开头，构成有序列表
  • 以-、*、+开头，构成无序列表
  • 嵌套列表：
    1.有序列表嵌套：在外层的基础上，缩进两格。然后从1开始
    2.无序列表嵌套：在外层的基础上，缩进两格。然后从-、*、+中选一个。注意：符号不同，会产生不同层。
    3.混合列表嵌套：在外层的基础上，缩进两格。然后使用另一种列表方式即可。

六、引用

  • 通过使用>来形成引用段落
  • 对应HTML中的<blockquote>
  • 引用可以是多个段落，但换行的中间必须也有>符号

七、表格

  • 每一列通过|来区分
  • 表头和表体通过需要用|---|---|来区分(这里是两列)
  • 表体之间通过换行来区分
  • 在第二行的分隔线---的左右两边通过加:来实现对齐效果。在左边加就是左对齐，右边就是右对齐，两边都加就是居中对齐
  • 在表格内部，可以使用其他元素

例子：

第一列 第二列 第三列
这一列是左对齐 这一列是居中对齐 这一列是右对齐
粗体 斜体 超链接

八、公式

  • 行内公式：使用一个$包裹公式的前后即可实现，在行内显示，不换行
    例子：
    $a^2 = b^2 + c^2$
  • 块公式：使用两个$包裹公式的前后即可实现，独占一行
    例子：
    $$
    f(x) = \int_{-\infty}^\infty
    \hat{f}(\xi),e^{2 \pi i \xi x},d\xi
    $$

  • 饼状图：pie

  • 序列图：sequenceDiagram
  • 类图：classDiagram
  • 状态图：stateDiagram
  • 用户旅程图：journey
